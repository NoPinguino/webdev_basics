Learn about breakpoints.

Breakpoints in Tailwind CSS are predefined screen-size thresholds that let you apply responsive styles with ease using simple prefix-based utility classes. Instead of writing custom media queries, you can attach breakpoint prefixes—like sm:, md:, lg:, xl:, or 2xl:—directly to your classes to control how elements behave at different viewport widths. Tailwind follows a mobile-first approach, meaning styles without prefixes apply to all screens, while breakpoint-prefixed classes take effect only at and above their respective sizes. This makes building responsive layouts both fast and intuitive, while keeping your CSS consistent and maintainable. Learn more abot the breakpoints: Tailwind documentation .

Breakpoint prefix Minimum width CSS
sm: 40rem (640px) @media (width >= 40rem) { ... }
md: 48rem (768px) @media (width >= 48rem) { ... }
lg: 64rem (1024px) @media (width >= 64rem) { ... }
xl: 80rem (1280px) @media (width >= 80rem) { ... }
2xl: 96rem (1536px) @media (width >= 96rem) { ... }

Movile first philosopy.

The mobile-first philosophy in Tailwind CSS means that styles are designed to work on small screens by default, and additional styles are layered on for larger screens using breakpoint prefixes. This approach ensures that the base experience is optimized for mobile devices, which are often the most common or constrained environment. By starting with mobile styles, developers can progressively enhance the layout and functionality for tablets, laptops, and desktops without duplicating CSS. In Tailwind, this is reflected in the way utility classes are applied: unprefixed classes target all screens, while classes with prefixes like md: or lg: override or extend these styles on larger viewports, promoting efficient, scalable, and responsive design.

Dark-mode page design.

In Tailwind CSS, the dark: pseudo-class allows you to define styles specifically for dark mode, enabling you to easily create a dark and light theme for your website. To use it, you first enable dark mode in your tailwind.config.js file—either by setting darkMode: 'media' to follow the user’s system preference, or darkMode: 'class' to toggle dark mode manually with a class. Once enabled, you can prefix any utility class with dark: to apply it only in dark mode, for example, bg-white dark:bg-gray-900 would make a background white in light mode and dark gray in dark mode. Combining this with a toggle button or a script that adds/removes the dark class on the html or body element allows you to build fully responsive pages that switch seamlessly between light and dark themes.

On this page, dark mode is implemented as a class on the tag. You can toggle it by clicking the button below.
